# Publication Matrix

## Recommended order

1. InfoQ
2. 阿里云开发者社区
3. 知乎

## Why this order

- InfoQ is best for the deep technical story: correctness gates, negative cases,
  and the kernel/fallback boundary.
- 阿里云开发者社区 is best for the practical reproduction package: DSW,
  ROCm, vLLM, notebook, figures, and repo links.
- 知乎 is best for broader reach and easier reading, but it is not the strongest
  credibility venue for this topic.

## Shared facts to keep

- DeepSeek-V4-Flash on AMD ROCm.
- vLLM OpenAI API server health.
- 2K / 8K / 32K needle retrieval.
- 8K `index_topk` sweep.
- negative KV cache planning case.
- no model weights in the public package.

## Tone differences

- InfoQ: deep, measured, technical, less first-person.
- 阿里云开发者社区: practical, reproducible, more implementation detail.
- 知乎: concise, readable, more problem-solution style.
